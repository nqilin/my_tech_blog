# 个人技术博客项目开发报告

# 摘要

本项目为个人技术博客网站，旨在搭建一个用于分享行业技术资料、原创研究报告、个人开发产品及开展学术讨论的线上平台。项目基于Python+Flask+SQLite技术栈，采用前后端分离的基础开发模式（原生前端+轻量后端），实现了文章展示、分类筛选、关键字搜索、后台管理（文章增删改查）、登录权限控制等核心功能。开发周期为1个月，全程从0搭建，完成本地开发、bug修复及线上免费部署，最终实现全网可访问、功能完整的个人技术博客，既满足技术分享需求，也提升了自身Web开发、数据库操作及项目部署的综合能力。

**关键词**：Web开发；Flask框架；SQLite数据库；技术博客；关键字搜索；Python

# 一、引言

## 1.1 开发背景

随着信息技术的快速发展，技术博客已成为程序员分享知识、积累成果、开展学术讨论的重要载体。作为大一程序员，搭建个人技术博客不仅能系统梳理所学技术、记录开发心得，还能通过分享行业资料、原创研究报告及个人开发产品，提升自身专业影响力，为后续学习和求职积累实践经验。当前市面上的博客平台多为通用型，难以贴合个人技术分享的个性化需求，因此决定从零开发一款专属个人的技术博客网站。

## 1.2 开发意义

1. **实践意义**：将课堂所学的Python、HTML/CSS/JavaScript等基础知识应用于实际项目，掌握Web开发的核心流程（前端渲染、后端接口开发、数据库交互、项目部署），提升问题解决能力和项目实战能力。
2. **实用意义**：搭建专属技术分享平台，实现行业资料、原创内容的规范化管理和全网展示，支持学术讨论和个人成果沉淀，为后续技术学习和交流提供支撑。
3. **成长意义**：熟悉Git版本控制、项目文档编写、线上部署等程序员必备技能，培养代码规范意识和项目管理思维，为后续毕业设计、进阶开发打下基础。

## 1.3 开发目标

### 1.3.1 功能目标

- 前台功能：实现文章列表展示、文章详情查看、分类筛选、标题/正文关键字搜索、关于我页面展示。
- 后台功能：实现管理员登录验证、文章发布/编辑/删除/查询、权限控制（未登录禁止访问后台）。
- 通用功能：数据实时同步、操作反馈提示、异常处理、页面互通无死链。

### 1.3.2 技术目标

- 掌握Flask轻量Web框架的核心用法，实现后端路由开发和前后端数据联动。
- 熟练使用SQLite数据库，完成数据增删改查及关键字搜索功能的实现。
- 掌握原生HTML/CSS/JavaScript的综合应用，完成前端页面开发和交互优化。
- 学会使用Git进行版本控制，完成PythonAnywhere免费线上部署，实现全网访问。

### 1.3.3 时间目标

总开发周期为1个月（4周），按阶段完成环境搭建、基础开发、核心功能实现、后台开发、部署上线及文档编写，每日投入1.5-2小时，确保项目按时交付且功能完整。

# 二、需求分析

## 2.1 用户需求

本项目用户分为两类：访客（普通用户）和博主（管理员），两类用户的核心需求如下：

### 2.1.1 访客需求

- 可浏览博客首页的所有技术文章，查看文章标题、分类、创建时间及完整正文。
- 可通过分类筛选，快速查找对应类别的技术文章（如行业资料、研究报告、原创产品）。
- 可通过关键字搜索，精准查找标题或正文中包含目标关键字的文章，提升查找效率。
- 可查看博主的个人信息，获取学术讨论的联系方式。

### 2.1.2 博主需求

- 可通过账号密码登录后台，未登录时无法访问后台任何功能，保障数据安全。
- 可在后台发布新文章，输入文章标题、分类、正文，提交后实时同步到前台。
- 可在后台编辑已发布的文章，修改后更新数据库，前台同步展示修改内容。
- 可在后台删除不需要的文章，删除后前台同步移除，避免冗余内容。
- 可在后台查看所有文章列表，快速进行编辑、删除操作，实现文章统一管理。

## 2.2 功能需求

### 2.2.1 前台核心功能

1. 页面展示：包含首页、文章详情页、分类页、关于我页，页面导航统一，支持互相跳转。
2. 文章展示：动态渲染数据库中的文章数据，展示标题、分类、创建时间、正文，支持文章详情查看。
3. 分类筛选：展示所有文章分类，点击分类筛选对应文章，实现分类精准查找。
4. 关键字搜索：首页和分类页均设置搜索框，支持标题/正文双维度匹配，无搜索结果时显示提示信息。

### 2.2.2 后台核心功能

1. 登录验证：提供专属登录页，支持账号密码验证，验证失败提示错误信息，成功进入后台。
2. 权限控制：未登录用户访问后台地址时，自动跳转到登录页，实现全局权限拦截。
3. 文章管理：实现文章的增删改查全操作，支持数据回显、操作反馈提示，避免误操作。

### 2.2.3 通用功能

1. 数据同步：后台对文章的操作（发布/编辑/删除）实时同步到前台，搜索结果同步更新。
2. 交互提示：表单提交、登录验证、文章操作均有弹窗提示，提升用户体验。
3. 异常处理：对数据库操作、URL参数、表单提交等场景进行异常拦截，避免程序崩溃。
4. 样式适配：本地运行和线上部署均保持样式统一，适配主流浏览器，支持基础移动端适配。

## 2.3 技术需求

- 前端：使用HTML搭建页面结构，CSS实现样式美化，JavaScript实现基础交互（表单验证、弹窗提示）。

- 数据库：使用SQLite文件型数据库，无需单独安装，实现文章数据的存储和查询。
- 版本控制：使用Git+GitHub，实现代码管理、版本控制和代码备份。

# 四、系统设计

## 4.1 系统总体设计

本系统采用前后端分离的基础架构（无框架分离，原生前端+Flask后端），整体分为前台展示层、后端服务层、数据存储层三个部分，各层职责清晰，协同工作实现系统所有功能。

### 4.1.1 前台展示层

负责向用户展示页面和接收用户操作，包括首页、文章详情页、分类页、关于我页、后台登录页及后台管理页面。通过HTML搭建页面结构，CSS美化样式，JavaScript实现交互，Jinja2模板接收后端数据并动态渲染。

### 4.1.2 后端服务层

基于Flask框架开发，是系统的核心层，负责处理前端请求、实现业务逻辑、调用数据库操作。主要包括路由开发、权限控制、表单数据处理、数据库函数调用、异常处理等功能，连接前台展示层和数据存储层。

### 4.1.3 数据存储层

采用SQLite数据库，负责存储系统所有数据，核心为article数据表，存储文章相关信息。通过Python的sqlite3模块实现数据的增删改查及关键字搜索操作，为后端服务层提供数据支撑，确保数据持久化存储。

## 4.2 数据库设计

### 4.2.1 数据库选型依据

本项目为小型个人博客，数据量小，无需高并发处理，SQLite作为文件型数据库，具备无需单独安装、配置简单、原生支持Python、操作便捷等优点，适配项目的开发和部署需求，无需投入额外精力维护数据库服务。

### 4.2.2 核心数据表设计

本项目仅需一张核心数据表（article），用于存储所有文章相关数据，无需额外设计关联表，简化数据库结构，降低开发难度。

| 字段名      | 数据类型 | 是否主键 | 是否非空 | 默认值   | 字段说明                                                     |
| ----------- | -------- | -------- | -------- | -------- | ------------------------------------------------------------ |
| id          | INTEGER  | 是       | 是       | 自增     | 文章唯一标识，用于区分不同文章，支撑编辑、删除、查询操作     |
| title       | TEXT     | 否       | 是       | 无       | 文章标题，用于前台展示和关键字搜索匹配                       |
| content     | TEXT     | 否       | 是       | 无       | 文章正文，存储技术资料、研究报告等核心内容，支持关键字搜索匹配 |
| category    | TEXT     | 否       | 是       | 技术笔记 | 文章分类，用于分类筛选，如“行业资料”“研究报告”“原创产品”     |
| create_time | TEXT     | 否       | 是       | 无       | 文章创建时间，格式为“YYYY-MM-DD”，用于前台展示               |

### 4.2.3 核心SQL语句

1. 创建article表：        `CREATE TABLE IF NOT EXISTS article ( ``    id INTEGER PRIMARY KEY AUTOINCREMENT, ``    title TEXT NOT NULL, ``    content TEXT NOT NULL, ``    category TEXT NOT NULL DEFAULT '技术笔记', ``    create_time TEXT NOT NULL ``);`
2. 关键字搜索语句（标题/正文双维度匹配）：        `SELECT * FROM article WHERE title LIKE ? OR content LIKE ?;`
3. 查询所有文章：        `SELECT * FROM article ORDER BY id DESC;`
4. 按ID查询文章：        `SELECT * FROM article WHERE id = ?;`
5. 新增文章语句：        `INSERT INTO article (title, content, category, create_time) VALUES (?, ?, ?, ?);`
6. 更新文章语句：        `UPDATE article SET title = ?, content = ?, category = ?, create_time = ? WHERE id = ?;`
7. 删除文章语句：        `DELETE FROM article WHERE id = ?;`

## 4.3 页面设计

### 4.3.1 页面总体风格

采用简洁、专业的技术风，以白色和深色为主色调，突出文字内容，避免冗余装饰，适配技术资料和研究报告的阅读需求；全站导航栏统一，页面布局规整，操作流程顺畅，提升用户体验。

### 4.3.2 核心页面结构设计

1. **首页**：顶部导航栏（博客标题、首页、分类、关于我）→ 搜索框（关键字输入+提交按钮）→ 文章列表（标题、分类、创建时间，点击标题进入详情）。
2. **文章详情页**：顶部导航栏→ 文章标题→ 文章信息（分类、创建时间）→ 文章正文→ 返回首页按钮。
3. **分类页**：顶部导航栏→ 搜索框→ 分类列表→ 对应分类文章列表。
4. **关于我页**：顶部导航栏→ 博主个人信息（专业、技术栈、开发初衷）→ 学术讨论联系方式。
5. **后台登录页**：账号输入框、密码输入框、登录按钮、错误提示区域。
6. **后台首页**：导航栏（后台首页、发布文章、退出登录）→ 文章列表（标题、分类、创建时间、编辑/删除按钮）。
7. **发布/编辑文章页**：标题输入框、分类输入框、正文输入框、提交按钮、返回后台按钮。

## 4.4 核心功能设计

### 4.4.1 关键字搜索功能设计

1. 前端：在首页和分类页添加搜索表单，用户输入关键字后提交，前端通过JS做非空验证，为空时弹窗提示并禁止提交。
2. 后端：接收前端提交的关键字参数，调用搜索查询函数，通过SQLite的LIKE语句实现标题/正文双维度匹配，获取匹配的文章数据。
3. 渲染展示：后端将匹配的文章数据传递到前端，通过Jinja2模板渲染到页面；若无匹配结果，显示“暂无相关文章”提示。

### 4.4.2 文章增删改查功能设计

1. 发布文章：前端提交表单数据→ 后端接收并做二次验证→ 调用新增函数存入数据库→ 弹窗提示发布成功→ 跳转到后台首页。
2. 编辑文章：点击编辑按钮→ 后端按ID查询文章数据并回显到表单→ 用户修改后提交→ 后端接收数据并更新数据库→ 弹窗提示更新成功→ 跳转到后台首页。
3. 删除文章：点击删除按钮→ 前端弹出确认弹窗→ 确认后后端按ID删除数据库数据→ 弹窗提示删除成功→ 刷新文章列表。
4. 查询文章：后台首页调用查询所有文章函数，前端渲染文章列表；前台首页/分类页根据需求调用查询或搜索函数，渲染对应文章。

### 4.4.3 权限控制功能设计

1. 登录验证：用户提交账号密码→ 后端比对写死的账号密码→ 验证成功则设置session标记登录状态→ 跳转到后台首页；验证失败则弹窗提示错误。
2. 全局权限拦截：封装登录验证装饰器，为所有后台路由添加装饰器→ 未登录用户访问后台时，装饰器拦截并跳转到登录页；已登录用户正常访问。

### 5.2.1 Flask搜索路由实现

```python
from flask import Flask, render_template, request
from utils.db_operate import search_articles, get_all_articles

app = Flask(__name__)
app.secret_key = 'your_secret_key'  # 用于session加密，自定义即可

# 首页路由（含搜索功能）
@app.route('/', methods=['GET'])
def index():
    # 获取搜索关键字，无则为None
    keyword = request.args.get('keyword')
    if keyword:
        # 调用搜索函数，实现标题/正文双维度匹配
        articles = search_articles(keyword)
    else:
        # 无关键字时，查询所有文章
        articles = get_all_articles()
    # 传递文章数据到前端，渲染页面
    return render_template('index.html', articles=articles, keyword=keyword)
    
```

### 5.2.2 数据库搜索函数实现

```python
import sqlite3
import os

# 数据库文件路径
DB_PATH = os.path.join(os.path.dirname(__file__), '../db/blog.db')

# 连接数据库
def connect_db():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row  # 便于通过字段名获取数据
    return conn

# 关键字搜索文章（标题/正文双维度）
def search_articles(keyword):
    conn = connect_db()
    cursor = conn.cursor()
    # LIKE语句实现模糊匹配，%表示任意字符
    cursor.execute('SELECT * FROM article WHERE title LIKE ? OR content LIKE ? ORDER BY id DESC',
                   (f'%{keyword}%', f'%{keyword}%'))
    articles = cursor.fetchall()  # 获取所有匹配结果
    conn.close()
    return articles
```

### 5.2.4 后台登录与权限控制实现

```python
from flask import session, redirect, url_for, flash, request
from functools import wraps

# 登录验证装饰器（权限控制核心）
def login_required(f):
    @wraps(f)
    def decorated_function(*args, **kwargs):
        # 检查session中是否有登录标记
        if 'logged_in' not in session:
            flash('请先登录后再操作！', 'error')
            return redirect(url_for('admin_login'))  # 未登录跳转到登录页
        return f(*args, **kwargs)
    return decorated_function

# 后台登录路由
@app.route('/admin/login', methods=['GET', 'POST'])
def admin_login():
    if request.method == 'POST':
        # 写死账号密码（基础版，后续可优化加密存储）
        username = request.form.get('username')
        password = request.form.get('password')
        if username == 'admin' and password == '123456':
            session['logged_in'] = True  # 设置登录标记
            flash('登录成功！', 'success')
            return redirect(url_for('admin_index'))  # 登录成功跳转到后台首页
        else:
            flash('账号或密码错误，请重新输入！', 'error')
    return render_template('admin/login.html')

# 后台首页路由（需登录验证）
@app.route('/admin')
@login_required  # 添加权限装饰器
def admin_index():
    articles = get_all_articles()
    return render_template('admin/admin_index.html', articles=articles)
```

## 5.3 开发过程中遇到的问题及解决方法

1. **问题1：Flask渲染HTML页面时提示“TemplateNotFound”错误**原因：HTML文件未放在Flask默认的templates文件夹中，或文件路径错误。解决方法：确认项目根目录存在templates文件夹，且HTML文件放在该文件夹下；渲染时使用render_template('文件名.html')，无需添加路径前缀。
2. **问题2：SQLite搜索功能无法匹配关键字，返回结果为空**原因：LIKE语句的参数传递错误，未添加%实现模糊匹配，仅能匹配完全一致的内容。解决方法：修改搜索函数，将关键字拼接为f'%{keyword}%'，确保模糊匹配标题和正文中包含关键字的内容。
3. **问题3：未登录用户可直接访问后台页面，权限控制失效**原因：未给所有后台路由添加登录验证装饰器，或session未正确设置。解决方法：为所有后台路由添加@login_required装饰器；确保app.secret_key已设置（用于session加密），登录成功后正确设置session['logged_in'] = True。
4. **问题4：线上部署后静态资源（CSS/JS）加载失败**原因：静态资源路径配置错误，PythonAnywhere未正确识别static文件夹。解决方法：前端引入静态资源时使用url_for('static', filename='css/style.css')；在WSGI配置文件中添加静态资源路径映射，确保路径正确。
5. **问题5：文章内容中的换行在页面中不显示，排版错乱**原因：HTML默认会忽略文本中的换行符，需要将换行符转换为<br/>标签。解决方法：在前端渲染文章正文时，使用Jinja2的replace过滤器，将\n转换为<br/>，代码为{{ art.content|replace('\n', '<br/>')|safe }}。
